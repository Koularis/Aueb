Αλλαγές σε σχέση με το πρώτο παραδοτέο:
-Πλέον ο consumer έχει ορισμένο ένα arraylist και ο publisher ενα ArrayBlockingQueue. Χρησιμεύουν στο να 
αποθηκεύουμε ότι πληροφορία από τον server ενδιαφέρει το thread της εφαρμογής και αντοίστοιχα για να αποθηκεύουμε
ότι input βγάζει η εφαρμογή σε ένα προσωρινό buffer.
-Πλέον ο publisher με τον consumer χρησιμοποιούν διαφορετικό socket.
-Μικρές αλλαγές που αφορούν την αποστολή και αποδοχή μηνυμάτων που θέλουμε να αποθηκεύσουμε στα buffers.

Οδηγίες:
-Το μόνο που χρειάζεται η εφαρμογή για να τρέξει είναι τα αρχεία από το android studio (xml,manifest,activities) και 
η κλάση του client. Όλα τα άλλα μπορούν να βρίσκονται σε διαφορετικό φάκελο (Topic,Profile,MFile,Server).
-Για να δουλέψει σωστά η εφαρμογή χρειάζεται το όνομα να έχει >= 5 γράμματα.
-Όταν γίνονται setup οι server πρέπει να ορίσουμε με την σειρά στο select Server τα νούμερα 0 1 και 2 (σε διαφορετικά processes).
